## Core Features
- Multi-chat threads: users can create multiple conversations and each thread stores its own messages.
- Shared chat snapshots: a chat can be shared with a signed link and opened as read-only.
- Conversation summary: each thread can be summarized with key topics and learned preferences.
- Emotion Pulse: frontend computes tone, trend, and energy from recent user messages.
- Privacy: chats are protected in transit over HTTPS and handled as private app data.

## Feature Locations (UI)
- New chat: left sidebar top button labeled "New Chat".
- Thread list: left sidebar middle area with conversation titles.
- Share: top-right header action button ("Share") in an active chat.
- Summarize: top-right header action button ("Summarize").
- Emotion Pulse:
  - "Show Emotion Pulse" button appears below the header when chat is open.
  - Pulse card appears in the main panel under header when enabled.
- Copy response: copy icon button directly below each assistant message bubble.
- Emoji picker: smile icon inside the message input row on the right.
- Theme toggle: moon/sun icon in the sidebar header.

## Sharing Rules
- Shared links open only one snapshot thread, not the owner's full thread list.
- Shared view is read-only to protect source conversation integrity and privacy.
- Shared viewers can read and copy content but cannot send, delete, or edit.
- If a share link is old/invalid, user must generate a new share link from current deployment.

## Trust and Safety
- Echo AI is supportive but not a doctor or therapist.
- In urgent safety situations, users should contact local emergency services or a trusted person nearby.
- The app avoids exposing internal infrastructure details during normal user chat.
